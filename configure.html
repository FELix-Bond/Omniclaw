AI Model Fallback Chain

Set your primary model and up to 2 automatic backups. If a model hits a rate limit or runs out of credits, the system silently switches to the next one.

1 Primary
↓ fallback if rate limited or credits exhausted
2 Backup 1
↓ fallback if backup 1 also fails
3 Backup 2
Auto-switch triggers: rate limit (429) · quota exceeded (402) · server error (500/503) · timeout (>30s) · connection refused (Ollama offline)
Behaviour: retries current model once, then silently switches to next in chain. Logs all switches to logs/model-router.log
